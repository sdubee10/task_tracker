## 📖 프로젝트 개요

### 프로젝트명
**TaskFlow** - 신청서 기반 업무 관리 및 분석 시스템

### 프로젝트 정의
TaskFlow는 기업 내 다양한 부서 간 업무 요청을 **통합 관리**하고, **그래프 데이터베이스**를 활용하여 신청서와 담당자 간의 관계를 **시각화**하며, 업무 처리 현황을 **데이터 기반으로 분석**할 수 있는 웹 기반 업무 관리 플랫폼입니다.

### 핵심 가치

| 가치 | 설명 | 구현 기능 |
|------|------|----------|
| 🎯 **효율성** | 직관적인 업무 요청 프로세스 | 드래그 앤 드롭 신청서 빌더, 3단계 작성 프로세스 |
| 📊 **가시성** | 업무 흐름의 시각적 파악 | 그래프 기반 관계 시각화, 6가지 레이아웃 |
| 📈 **분석력** | 데이터 기반 의사결정 지원 | 개인/팀별 업무량 분석, 역량 평가, 처리 현황 통계 |
| 🔄 **유연성** | 다양한 업무 유형 지원 | 25종 컴포넌트, 30종 템플릿, 커스텀 템플릿 |
| 🎨 **사용자 경험** | 편리한 인터페이스 | 6가지 테마, 반응형 디자인, SPA 구조 |
| 🔗 **추적성** | 업무 이력 관리 | 상태 변경 히스토리, 담당자 배정 기록, 평가 시스템 |

## 🎯 기획 배경 및 목적

### 기획 배경

기업 환경에서 발생하는 다양한 업무 요청(개발 요청, 데이터 추출, 인프라 설정, 보안 점검 등)은 다음과 같은 문제점을 가지고 있습니다:

#### 문제점 분석

| 문제 | 현황 | 영향 |
|------|------|------|
| **비표준화된 요청 양식** | 부서마다 다른 양식, 이메일/메신저 등 산발적 요청 | 정보 누락, 커뮤니케이션 비용 증가 |
| **업무 현황 파악 어려움** | 누가 어떤 업무를 처리 중인지 파악 불가 | 중복 업무, 병목 현상 |
| **업무량 불균형** | 특정 담당자에게 업무 집중 | 번아웃, 처리 지연 |
| **데이터 분석 한계** | 업무 처리 현황에 대한 정량적 분석 부재 | 의사결정 근거 부족 |
| **유연하지 않은 데이터 구조** | 신청서마다 다른 컴포넌트로 관계형 DB 한계 | 시스템 확장성 제한 |
| **업무 평가 체계 부재** | 처리 품질에 대한 피드백 시스템 없음 | 서비스 품질 관리 어려움 |

### 기획 목적

#### 1. 통합 신청서 시스템 구축
- 모든 부서의 업무 요청을 하나의 플랫폼에서 관리
- 표준화된 템플릿으로 일관된 요청 프로세스 확립

#### 2. 시각적 업무 관리
- 그래프 기반 시각화로 업무 관계 및 현황 실시간 파악
- 담당자-신청서 간 연결 관계 직관적 표현

#### 3. 데이터 기반 의사결정 지원
- 업무량, 처리 시간, 완료율 등 정량적 데이터 제공
- 개인/팀별 역량 평가 및 업무 특성 분석

#### 4. 유연한 템플릿 시스템
- 각 부서 특성에 맞는 신청서 양식 자유롭게 설계
- 25종 컴포넌트로 다양한 입력 형태 지원

#### 5. 그래프 데이터베이스 도입
- 신청서별 다양한 컴포넌트 구성을 유연하게 저장
- 관계 중심 데이터 모델로 복잡한 업무 관계 표현

#### 6. 업무 품질 관리 체계
- 요청자의 처리 결과 평가 시스템
- 기술적/정성적/정량적 평가로 서비스 품질 향상

## 📋 상세 사용 시나리오

### 시나리오 1: 신청서 템플릿 생성 (관리자/팀장)

#### 배경
DBA팀 팀장 김철수는 팀에서 자주 받는 "데이터 마이그레이션 요청"에 대한 표준화된 신청서 양식을 만들고자 합니다.

#### 시나리오 흐름

```
┌─────────────────────────────────────────────────────────────────────┐
│ Step 1: 요청서 만들기 접속                                            │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 사이드바에서 "요청서 만들기" 클릭                                    │
│ 2. 추천 템플릿 또는 빈 캔버스로 시작                                    │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 2: 컴포넌트 배치                                                 │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 왼쪽 팔레트에서 "섹션 제목" 드래그 → 캔버스에 드롭                    │
│ 2. "텍스트 입력" 컴포넌트 추가 (마이그레이션 대상 테이블)                 │
│ 3. "텍스트 영역" 컴포넌트 추가 (마이그레이션 사유)                       │
│ 4. "날짜 선택" 컴포넌트 추가 (희망 완료일)                              │
│ 5. "우선순위 선택" 컴포넌트 추가                                        │
│ 6. "파일 첨부" 컴포넌트 추가 (관련 문서)                                │
│ 7. 하단 드롭 영역에 추가 컴포넌트 배치                                   │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 3: 컴포넌트 속성 설정                                            │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 각 컴포넌트 클릭 → 우측 속성 패널에서 설정                           │
│    - 레이블: "마이그레이션 대상 테이블명"                                │
│    - 플레이스홀더: "예: USER_INFO, ORDER_HISTORY"                      │
│    - 필수 여부: 체크                                                   │
│ 2. 컴포넌트 너비 조정 (1열, 2열, 전체)                                  │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 4: 미리보기 및 저장                                              │
├─────────────────────────────────────────────────────────────────────┤
│ 1. "미리보기" 버튼 클릭 → 실제 신청서 형태 확인                         │
│ 2. "저장" 버튼 클릭                                                    │
│ 3. 템플릿 정보 입력:                                                   │
│    - 이름: "데이터 마이그레이션 요청서"                                  │
│    - 설명: "테이블 간 데이터 마이그레이션 요청 시 사용"                   │
│    - 카테고리: "DBA"                                                   │
│ 4. 저장 완료 → "내 템플릿"에서 확인 가능                                │
└─────────────────────────────────────────────────────────────────────┘
```

#### 기대 효과
- 표준화된 요청 양식으로 필요한 정보 누락 방지
- 팀 내 일관된 업무 프로세스 확립
- 신규 입사자도 쉽게 사용 가능

### 시나리오 2: 신청서 작성 및 제출 (요청자)

#### 배경
마케팅팀 박영희는 캠페인 분석을 위해 DBA팀에 데이터 추출을 요청하고자 합니다.

#### 시나리오 흐름

```
┌─────────────────────────────────────────────────────────────────────┐
│ Step 1: 템플릿 선택                                                   │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 사이드바에서 "신청서 작성" 클릭                                      │
│ 2. 카테고리 탭에서 "DBA" 선택                                          │
│ 3. "데이터 추출 요청서" 템플릿 클릭                                     │
│ 4. "다음" 버튼 클릭                                                    │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 2: 신청서 내용 작성                                              │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 동적으로 생성된 폼에 정보 입력:                                      │
│    - 요청 제목: "2024년 1분기 캠페인 성과 데이터 추출"                   │
│    - 추출 목적: "마케팅 캠페인 ROI 분석"                                │
│    - 대상 테이블: "CAMPAIGN_RESULT, USER_CONVERSION"                   │
│    - 추출 조건: "기간: 2024-01-01 ~ 2024-03-31"                        │
│    - 출력 형식: "Excel (.xlsx)"                                        │
│    - 우선순위: "높음"                                                  │
│    - 희망 완료일: "2024-04-10"                                         │
│ 2. "AI 검증" 버튼 클릭 (선택사항)                                       │
│    → AI가 신청서 내용 검토 및 피드백 제공                               │
│ 3. "다음" 버튼 클릭                                                    │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 3: 검토 및 제출                                                  │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 입력 내용 최종 확인                                                 │
│ 2. 수정 필요 시 "이전" 버튼으로 돌아가기                                │
│ 3. "제출" 버튼 클릭                                                    │
│ 4. 성공 모달 표시:                                                     │
│    - "신청서가 성공적으로 제출되었습니다!"                               │
│    - 신청서 ID: REQ-2024-0412-001                                      │
│    - "대시보드로 이동" 또는 "새 신청서 작성" 선택                        │
└─────────────────────────────────────────────────────────────────────┘
```

#### 기대 효과
- 3단계 프로세스로 실수 없는 신청서 작성
- AI 검증으로 불완전한 신청서 사전 방지
- 즉각적인 제출 확인 및 추적 가능

### 시나리오 3: 신청서 접수 및 처리 (담당자)

#### 배경
DBA팀 이민수는 자신에게 배정된 데이터 추출 요청을 처리합니다.

#### 시나리오 흐름

```
┌─────────────────────────────────────────────────────────────────────┐
│ Step 1: 신청서 목록 확인                                              │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 사이드바에서 "신청서 목록" 클릭                                      │
│ 2. "내 신청서" 필터에서 "내가 담당자인" 선택                            │
│ 3. 또는 섹션 뷰에서 "내가 처리 중인 신청서" 확인                         │
│ 4. 새로 들어온 신청서 클릭                                             │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 2: 신청서 상세 확인 및 접수                                       │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 신청서 상세 페이지에서 요청 내용 확인                                │
│    - 요청자 정보, 요청 내용, 첨부 파일 등                               │
│ 2. 처리 가능 여부 판단                                                 │
│ 3-A. 접수 가능한 경우:                                                 │
│    → "접수" 버튼 클릭                                                  │
│    → 상태가 "처리 중"으로 변경                                          │
│ 3-B. 추가 정보 필요한 경우:                                            │
│    → "반려" 버튼 클릭                                                  │
│    → 반려 사유 입력                                                    │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 3: 업무 처리 및 완료                                             │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 요청 내용에 따라 실제 업무 수행                                      │
│    (예: SQL 쿼리 작성 및 실행, 데이터 추출)                             │
│ 2. 처리 완료 후 "완료" 버튼 클릭                                        │
│ 3. 상태가 "완료"로 변경                                                │
│ 4. 요청자에게 완료 알림 (향후 구현)                                     │
└─────────────────────────────────────────────────────────────────────┘
```

#### 추가 기능: 담당자 배정/변경

│    → 다중 담당자 배정 가능                                              │
│                                                                       │
│ 3. 담당자 변경/제거                                                    │
│    → 담당자 목록에서 X 버튼 클릭하여 제거                               │
└─────────────────────────────────────────────────────────────────────┘
```

### 시나리오 4: 업무 처리 평가 (요청자)

#### 배경
마케팅팀 박영희는 DBA팀에서 처리 완료된 데이터 추출 요청에 대해 업무 품질을 평가합니다.

│ 3. 처리 결과 확인                                                     │
└─────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────┐
│ Step 2: 업무 처리 평가                                                │
├─────────────────────────────────────────────────────────────────────┤
│ 1. "업무 처리 평가하기" 버튼 클릭                                       │
│ 2. 평가 모달에서 각 항목 평가:                                          │
│                                                                       │
│    [정량적 평가]                                                       │
│    - 전체 만족도: ⭐⭐⭐⭐⭐ (5점)                                      │
│    - 처리 속도: ⭐⭐⭐⭐☆ (4점)                                        │
│    - 결과 품질: ⭐⭐⭐⭐⭐ (5점)                                        │
│    - 커뮤니케이션: ⭐⭐⭐⭐☆ (4점)                                      │
│                                                                       │
│    [기술적 평가]                                                       │
│    - 전문성: ✓ 높음                                                   │
│    - 정확성: ✓ 매우 정확                                               │
│    - 효율성: ✓ 효율적                                                  │
│                                                                       │
│    [정성적 평가]                                                       │
│    - 상세 피드백: "요청한 데이터가 정확하게 추출되었고,                   │

│                                                                       │
│    [재요청 의향]                                                       │
│    - ◉ 매우 그렇다 ○ 그렇다 ○ 보통 ○ 아니다                          │
│                                                                       │
│ 3. "평가 제출" 버튼 클릭                                               │
└─────────────────────────────────────────────────────────────────────┘
```

#### 기대 효과
- 업무 품질에 대한 정량적/정성적 피드백 수집
- 담당자 역량 평가 데이터 축적
- 서비스 품질 지속적 개선

| 기능 | 설명 |
|------|------|
| 통계 카드 | 총 신청서, 완료, 처리 중, 대기 건수 |

### 3. 신청서 목록 (`#/requests`)

| 기능 | 설명 |
|------|------|
| 섹션 뷰 | 카테고리별 섹션으로 구분 (접기/펼치기) |
| 테이블 뷰 | 전체 신청서 테이블 형태 |
| 검색 | ID, 제목, 요청자 검색 |
| 필터 | 내 신청서, 상태, 우선순위, 카테고리 |
| 정렬 | 컬럼 헤더 클릭으로 정렬 |
| 새로고침 | 토스트 알림과 함께 데이터 갱신 |

### 4. 신청서 상세 (`#/request/:id`)

| 기능 | 설명 |
|------|------|
| 기본 정보 | ID, 카테고리, 우선순위, 마감일 |
| 요청자 정보 | 이름, 팀, 이메일 |
| 상세 내용 | 템플릿 컴포넌트별 입력 데이터 |
| 담당자 관리 | 내가 담당하기, 담당자 지정/제거 |
| 상태 변경 | 접수, 반려, 완료 |
| 처리 이력 | 상태 변경 히스토리 |
| 평가 | 요청자의 업무 처리 평가 |

### 5. 신청서 작성 (`#/request-form`)

| 기능 | 설명 |
|------|------|
| Step 1 | 템플릿 선택 (카테고리별 탭) |
| Step 2 | 동적 폼 작성, AI 검증 |
| Step 3 | 검토 및 제출 |
| 성공 모달 | 제출 완료 안내, 신청서 ID 표시 |

### 6. 요청서 만들기 (`#/form-builder`)

| 기능 | 설명 |
|------|------|
| 컴포넌트 팔레트 | 25종 컴포넌트 드래그 앤 드롭 |
| 캔버스 | 실시간 미리보기, 드롭 영역 |
| 속성 패널 | 컴포넌트 설정 |
| 추천 템플릿 | 상단 10개 |
| 기타 템플릿 | 카테고리별 분류 (접기/펼치기) |
| 내 템플릿 | 4열 그리드 |
| 미리보기 | 실제 신청서 형태 확인 |
| 저장 | 이름, 설명, 카테고리 설정 |

### 7. 업무 분석 (`#/analytics`)

| 탭 | 기능 |
|-----|------|
| 전체 현황 | 요약 카드, 일별 추이, 카테고리/우선순위/상태별 차트 |
| 업무량 분석 | 부서별 카테고리 → 팀 선택 → 팀원 노드 → 개인 분석 |
| 역량 평가 | 담당자별 처리 건수, 평균 시간, 평가 점수, 강점/개선점 |
| 팀별 통계 | 팀별 카드, 그래프 DB 분석 (부서별 처리량, Top 10 담당자, 매트릭스) |

### 8. 신청서 관계 (`#/graph`)

| 기능 | 설명 |
|------|------|
| 그래프 시각화 | SVG 기반 노드/엣지 렌더링 |
| 레이아웃 | 자유, 계층형, 원형, 그리드, 부서별, 상태별 |
| 필터 | 상태, 부서, 기간 (오늘~1년, 직접 선택) |
| 줌/패닝 | 마우스 휠, 드래그 |
| 노드 상세 | 우측 패널에 상세 정보 |
| 노드 추가 | 신청서/담당자 추가 모달 |

## 💾 데이터베이스 설계

### 그래프 데이터베이스 구조

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Request   │     │   Member    │     │    Team     │
│  (신청서)   │────▶│  (담당자)   │────▶│    (팀)     │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │                   │
       │                   │                   ▼
       │                   │            ┌─────────────┐
       │                   │            │ Department  │
       │                   └───────────▶│   (부서)    │
       │                                └─────────────┘
       │
       ▼
┌─────────────┐     ┌─────────────┐
│ Evaluation  │     │   History   │
│   (평가)    │     │  (히스토리) │
└─────────────┘     └─────────────┘
```

### 노드 타입

| 노드 타입 | 설명 | 주요 속성 |
|-----------|------|-----------|
| `request` | 신청서 | id, title, status, priority, category, formData |
| `member` | 담당자 | id, name, department, team, role, email |
| `team` | 팀 | id, name, departmentId |
| `department` | 부서 | id, name |
| `evaluation` | 평가 | technical, communication, efficiency, quality |

### 엣지 타입

| 엣지 타입 | 방향 | 설명 |
|-----------|------|------|
| `ASSIGNED_TO` | Request → Member | 담당자 배정 |
| `REQUESTED_BY` | Request → Member | 요청자 |
| `BELONGS_TO` | Member → Team | 팀 소속 |

## 🚀 시작하기

### 요구사항

- 모던 웹 브라우저 (Chrome, Firefox, Safari, Edge)
- Python 3.x 또는 Node.js (로컬 서버용)

### 설치 및 실행

```bash
# 1. 프로젝트 다운로드
git clone <repository-url>
cd task_tracker_v2

# 2. 로컬 서버 실행
python3 -m http.server 8080
# 또는
npx serve -p 8080

# 3. 브라우저에서 접속
# http://localhost:8080/login.html
```
